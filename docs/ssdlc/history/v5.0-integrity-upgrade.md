# SSDLC v5.0: 完整性防禦與解釋漂移預防 (Major Integrity Upgrade)

## 核心目標
防止 AI 在開發過程中，將「具體的業務細節」逐漸「摘要化/模糊化」為「通用的技術描述」，從而導致交付物與原始需求產生嚴重的 Interpretation Drift (解釋漂移)。

## 變更紀錄
1. **Source Intent Inventory (Step 3)**: 在啟動前強制提取「具體名詞」清單（Actor, Dependency, Constraint）。
2. **Controlled Status Vocabulary**: 嚴格規定狀態詞彙：`implemented-live`, `simulated-only`, `validation-only`。嚴禁使用 "Done" 或 "Finished" 等模糊字眼。
3. **Critical Intent Contract**: Phase 0 產出的合約將作為整個生命週期的真相來源。
4. **Task Decomposition Rules**: 規定任務必須細化到無法被不同團隊解讀為不同結果的程度。

## 解決的痛點
- AI 宣稱開發完成，實際卻只是寫了 UI 外殼而沒有串接正式 API。
- 專用整合（如 Stripe）被 AI 偷換為通用支付介面。
