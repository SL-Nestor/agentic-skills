# SSDLC v7.3: Skill-Enterprise 同步與 Hotfix 快車道

## 核心目標
解決第二輪深度審計發現的「主文件與 Lifecycle Skill 之間的認知斷層」—— 所有 6 個 Lifecycle Skill 現在都具備 Enterprise Mode Override 段落，消除了 Enterprise 規則在實際執行時被系統性跳過的風險。

## 變更紀錄

### P0 修正（立即修復）
1. **Step 編號重複**：修正啟動指令中的兩個 Step 3，正確重新編號為 Step 1-9。
2. **HTML 註解殘留**：移除 4 處無用的 HTML 註解區塊，節省 Token。

### P1 修正（Lifecycle Skill Enterprise 同步）
3. **lifecycle-spec**：加入 Enterprise Mode Override（輸入來源為 GOV 交付包、不設計 API、Gap Report）。
4. **lifecycle-plan**：加入 Enterprise Mode Override（Contract-Adherence 而非 Contract-First、驗收追溯）。
5. **lifecycle-build**：加入 Enterprise Mode Override（合約遵循、模組邊界、YAML metadata）+ Standards 交叉引用（Dependency Gate、Migration Safety、Concurrency Policy）。
6. **lifecycle-verify**：加入 Enterprise Mode Override（合約端點覆蓋率、效能預算）。
7. **lifecycle-ship**：加入 Enterprise Mode Override（Contract Drift Check 用 oasdiff、交接表更新、交付報告）。
8. **lifecycle-debug**：加入 Enterprise Mode Override（合約感知除錯、模組範圍限制）。

### P1 修正（新功能）
9. **Hotfix 快車道 (`--hotfix`)**：對齊 GOV-004 §12.1 緊急修正條款。跳過 Phase 0-4，直接進入 Phase 5 Build，強制 Prove-It 迴歸測試，Ship 時補回 Spec 與 Writeback。
10. **Artifact Mapping 雙表化**：將 Section 5 從單一 OpenSpec 表改為 Agile / Enterprise 雙表對照，消除 Enterprise 模式下 AI 對 artifact 名稱的困惑。

### P2 修正（防禦性改善）
11. **Agile→Enterprise 邊界偵測**：在 Phase 5-6 Build 中加入自動檢查，若 Agile 模式下目標路徑位於 `src/modules/`，主動提醒使用者。
12. **init-enterprise-repo**：加入 ENG-only 明確標註，EXP/REQ 倉引導使用 GOV 官方腳手架。

## 結果
- **Skill 自成閉環度**：從 ⭐3 恢復至 ⭐5（所有 Skill 現在都能在不依賴主文件的情況下正確執行 Enterprise 邏輯）。
- **治理整合**：從 ⭐4 升至 ⭐5（Hotfix 快車道對齊 GOV-004 §12.1，Contract Drift Check 對齊 GOV-019）。
