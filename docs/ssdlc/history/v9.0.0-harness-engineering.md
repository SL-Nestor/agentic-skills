# v9.0.0 - Harness Engineering Reinforcement (Context Engineering + Structured Handoff + Agent Resilience)

**Date**: 2026-04-30
**Author**: Antigravity (AI Architect)
**Reference**: [Harness Engineering Guide](https://harness-guide.com) + [nexu-io/harness-engineering-guide](https://github.com/nexu-io/harness-engineering-guide)

## 📌 變更背景 (Motivation)

v8.0.0 建立了 Universal Handoff Protocol，成功解決了 Token 爆炸與上下文漂移問題。但在實際運行中，仍存在以下進化空間：

1. **交接單被 AI 改寫**：`latest_memo.md` 是 Markdown 散文，LLM 傾向「美化」或「重寫」進度描述，導致下一個 Agent 對任務完成度產生錯覺（Premature Victory）。
2. **缺少 Token 預算管理**：`copilot-instructions.md` 每次自動載入 ~8K tokens，加上 Agent persona + tool schemas，開場就消耗 ~15K-20K tokens，沒有系統化的優先級控制。
3. **Agent 卡在迴圈**：沒有 Turn Budget 或 Loop Detection，Agent 遇到反覆失敗時會無限重試，浪費 Token 直到 context 爆炸。
4. **缺少跨 Sprint 記憶**：每次新 Session 只能載入靜態指令（copilot-instructions），無法讀取「上次這個專案踩過什麼坑」。

## 🎯 核心決策 (Core Decisions)

基於 Harness Engineering Guide 的四大支柱（Agentic Loop、Tool System、Memory & Context、Guardrails），我們執行了 7 項 Gap Analysis 並實作了其中 6 項。

### 1. 雙軌交接機制 (Dual-Track Handoff) — GAP-4 [P0]
- 新增 `templates/latest_state.json` 結構化狀態模板
- Agent 在每個 Gate 同時產出 `latest_memo.md`（散文）+ `latest_state.json`（JSON SSOT）
- JSON 中只有 `passes` 欄位可由 Agent 修改，其餘為 read-only
- **原理**：LLM 對 JSON 的處理行為是「讀寫特定欄位」，不會像 Markdown 那樣「隨意重寫」

### 2. Token 預算管理 (Context Budget) — GAP-1 [P0]
- 新增 `standards/context-budget.md`
- 定義 5 層載入優先級（Layer 0-5）
- 定義 Skill 載入協議：Menu → Select → Load → Execute → Unload（同時最多 2 Skill）
- 定義 Auto-Handoff 閾值：60% 警告 → 80% 強制交接 → 90% 硬停

### 3. 錯誤處理協議 (Error Handling Protocol) — GAP-3 [P1]
- 新增 `standards/error-handling-protocol.md`
- Turn Budget：每個 Skill 最多 25 turns，15 turns 警告
- Loop Detection：同一操作連續 3 次無進展 → 強制停止
- Tool Error Propagation：2-strike 規則
- Graceful Degradation：4 級降級策略（重試 → 部分完成 → 換模型 → 人類介入）
- Missing Capability Principle：「不要只修 bug，要修產生 bug 的系統」

### 4. 四層記憶架構 (Tiered Memory) — GAP-6 [P1]
- 新增 `templates/MEMORY.md` 專案長期記憶模板
- 定義 Tier 0-3 記憶分層：靜態指令 → Session 交接 → 專案記憶 → 日誌

### 5. Skill 標準化 (Skill Template) — GAP-2 [P2]
- 新增 `standards/skill-template.md`
- 強制所有 SKILL.md 包含：When to Use、Process、Anti-Rationalization、Exit Criteria

### 6. 機器可讀輸出標準 (Machine-Readable Output) — GAP-7 [P2]
- 擴展 `harness-engineering.md` 加入 §6 Agent-Readable Output Standards
- 統一狀態標記：`[PASS]` / `[FAIL]` / `[ERROR]` / `[WARN]` / `[SKIP]`
- Tracker 加入 HTML comment metadata（`<!-- status:PASS agent:spec-architect -->`）

## 🛡️ 影響評估
- **向前相容**：完全向前相容。v8.0 的 `latest_memo.md` 機制不變，`latest_state.json` 為新增的平行軌道。
- **Token 節省**：Context Budget 標準化後，預估每次對話可省 ~5K tokens 基礎消耗。
- **可靠性提升**：Turn Budget + Loop Detection 防止 Agent 無限迴圈燒錢。
- **未實作項目**：GAP-5（Fan-Out/Fan-In 並行執行）延後至 v10.0，目前 Sequential Pipeline 已足夠。
