# SSDLC v9.2.1: 技能預算最佳化 (Skills Context Budget Optimization)

## 核心目標
解決 Codex CLI 啟動時 91 個技能的 frontmatter (~45KB) 超過 2% skills context budget 的問題，
將技能分為「核心」與「擴充庫」兩層目錄架構。

## 變更紀錄

### 目錄重組
```
.agents/
├── skills/           ← 核心技能 (15 個)，Codex 啟動時自動掃描
│   ├── lifecycle-*   ← SSDLC 生命週期 (6)
│   ├── team-coordinator, requirement-analyst, spec-architect
│   ├── code-review-expert, security-gate, db-architect
│   ├── devops-eng, qa-tester, tech-writer
│
└── skills-library/   ← 擴充技能庫 (76 個)，按需載入
    ├── dotnet/        ← .NET 專屬 (26)
    ├── build/         ← 建構/MSBuild (12)
    ├── frontend/      ← 前端開發 (11)
    ├── platform/      ← 平台整合 (10)
    └── meta/          ← 元技能/工具 (17)
```

### 文件更新
| 檔案 | 變更 |
|------|------|
| `payload/AGENTS.md` | 新增完整的擴充技能索引表 (76 個技能的 macro → path 對照) |
| `payload/CLAUDE.md` | 新增 Extended Skills Library 段落，指引 Claude Code 查找擴充庫 |
| `payload/.github/copilot-instructions.md` | §0.4 新增 Two-Tier Skill Discovery 說明 |

### 預期效果
- Codex CLI 啟動掃描的 frontmatter 從 ~45KB (91 skills) 降至 ~7.5KB (15 skills)
- 遠低於 2% context budget 上限，不再出現 "descriptions were shortened" 警告
- 擴充技能透過 AGENTS.md 索引表按需載入，功能完全保留
