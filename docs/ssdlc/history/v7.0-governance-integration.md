# SSDLC v7.0: 企業治理架構整合與雙軌制 (Governance Integration)

## 核心目標
終結「敏捷開發框架」與「企業嚴格治理」之間的衝突。引入雙軌制 (Dual-Track)，在保留一般專案高速開發能力的同時，滿足跨模組、強依賴核心系統對「單一真相 (SSOT)」的嚴格規範。

## 變更紀錄
1. **雙軌制發動 (Dual-Track System)**:
   - 更新啟動指令，支援 `--enterprise` 標籤。
   - 一般模式：維持現狀，直接以 PRD 為 SSOT 高速開工。
   - 企業模式：啟動前強制要求提供 `tpl_req_eng_handoff_checklist.md`，並鎖定 API 合約基準 (`contract_baseline_ref`)。
2. **OpenAPI 絕對主權與回寫機制 (Writeback Rule)**:
   - 在企業模式下，AI 不再盲信 PRD 內的技術描述。OpenAPI 才是唯一的王。
   - 若開發過程需要修改 API 欄位或狀態，AI 被「嚴格禁止」直接修改程式碼，必須暫停並發起對 REQ 倉庫的合約建議修改 (GOV-004)。
3. **大倉模組邊界 (Monorepo Bounds)**:
   - 將程式碼產生範圍嚴格限制於 `src/modules/<module_name>` 中，配合 CODEOWNERS 避免多人開發衝突。
4. **統一開倉技能 (Init Enterprise Repo)**:
   - 引入 `$init-enterprise-repo` 技能，協助新工程倉與需求倉 (REQ) 建立元數據連結。
5. **配套治理範本 (Governance Templates)**:
   - 產出 `tpl_req_eng_handoff_checklist.md`、`tpl_formal_prd.md`、`tpl_acceptance_spec.md` 與對照指引，完整彌合需求端與工程端的邊界認知。

## 解決的痛點
- 解決大型系統（如：e首發票 20+ 模組）因各自開發導致合約分裂的問題。
- 讓 SSDLC 真正成為能走進大型企業的可靠執行力，而不只是一個「會寫 code 的玩具」。
