# SSDLC v9.1: Codex 雙軌開發支援 (Codex Dual-Mode Support)

## 核心目標
新增 OpenAI Codex 平台的原生支援，涵蓋 **VSCode Codex Extension** 與 **Codex CLI** 兩種開發模式，使 SSDLC 成為業界覆蓋率最完整的多代理配置框架。

## 變更紀錄
1. **新增 `AGENTS.md`**: 在 `payload/` 目錄下新增 Codex 統一入口檔（`AGENTS.md`），遵循 Codex 的標準指令發現機制（hierarchical `AGENTS.md` discovery）。
2. **Codex 專屬指引**: 加入 Sandbox Awareness、File System Discipline、Turn Budget、Approval Policy 等 Codex 特有的行為守則。
3. **安裝腳本更新**: `install.ps1` 與 `install.sh` 已同步更新，部署時自動複製 `AGENTS.md` 至目標專案根目錄。

## 支援平台一覽（v9.1 後）
| 平台 | 入口檔 | 狀態 |
|------|--------|------|
| GitHub Copilot / Cline | `.github/copilot-instructions.md` | ✅ 核心 SSOT |

| Gemini CLI / Vertex AI | `.geminirules` | ✅ 已支援 |
| Claude Code / Anthropic | `CLAUDE.md` | ✅ 已支援 |
| **OpenAI Codex (VSCode + CLI)** | **`AGENTS.md`** | **✅ 新增** |

## 技術細節
- Codex 使用 `AGENTS.md` 作為標準指令檔案名稱，支援 hierarchical discovery（Global `~/.codex/AGENTS.md` → Project Root → Subdirectory）。
- VSCode Codex Extension 與 Codex CLI 共用同一個 `AGENTS.md`，無需分別維護。
- 所有入口檔均採用「薄代理」設計：自身僅包含 Karpathy Directive 與載入指令，真正的 SSDLC 邏輯集中在 `.github/copilot-instructions.md`（Single Source of Truth）。
