# SSDLC v9.2: 主力引擎切換 (Primary Engine Shift)

## 核心目標
正式將開發主力引擎從 GitHub Copilot / Cline 切換至 **OpenAI Codex (VSCode Extension + CLI)** 與 **Claude Code**，反映團隊實際的開發工具鏈轉型。

## 變更紀錄

### 戰略調整
1. **主力引擎 (Primary)**：OpenAI Codex (VSCode + CLI) 與 Claude Code
2. **相容引擎 (Secondary)**：Cursor (`.cursorrules`)、Gemini CLI (`.geminirules`)
3. **SSOT 維持不變**：`.github/copilot-instructions.md` 保留作為 SSDLC 核心協議文件（Single Source of Truth），所有入口檔統一導向此文件。檔名中的 "copilot" 僅為歷史慣例，不代表它專屬於 Copilot。

### 文件更新
| 檔案 | 變更 |
|------|------|
| `README.md` | 版本升至 v9.2.0，開頭描述改為 Codex + Claude 主力引擎，部署說明重新排序 |
| `docs/USER_MANUAL.md` | 啟動方式重整為「主力 / 備用」兩層架構，Agent 職責表更新引擎標示 |
| `payload/.github/copilot-instructions.md` | 版本升至 v9.2.0，header comment 更新，Changelog 新增 v9.2.0 紀錄 |

### 不變項目
- `payload/AGENTS.md`：無需變更（v9.1.0 已建立）
- `payload/CLAUDE.md`：無需變更（原已存在）
- `payload/.cursorrules`：保留相容（薄代理）
- `payload/.geminirules`：保留相容（薄代理）
- `install.ps1` / `install.sh`：無需變更（v9.1.0 已更新部署清單）
