# SSDLC v7.5: 多核心引擎切換 (The Four-Core Activation)

## 核心目標
為了解決「用殺雞牛刀處理小需求」的反模式，將單一且冗長的 `/start-ssdlc` 啟動指令，重構成能對應不同專案規模的 **四核驅動模式**。

## 變更紀錄

### 1. 廢除舊有指令 CLI 化
- 移除了冗長且難記的 `/start-ssdlc <Target> [--mode=...] [--enterprise] [--hotfix]`。
- 將原本複雜的分支邏輯抽離，改以四個直觀的 Macro Commands 啟動。

### 2. 定義四核工作模式
在 `copilot-instructions.md` 第 0.5 節正式實作了分流：
1. **🏢 `$enterprise` (企業模式)**：繼承原有的 `--enterprise` 標籤，強制對著合約與治理倉範本開發。
2. **📝 `$agile` (規格開發模式)**：標準的 SSDLC 流程，PRD 即 SSOT，跑完完整的 10 個 Phase。
3. **🪶 `$light` (輕量開發模式)**：專門為雛形開發、小應用而設。拔除了 Tracker 和繁重的 Gate 審核，專注於簡單直接的 TDD。
4. **🧰 `$legacy` 或 `$tactical` (維護模式)**：跳脫 SSDLC，強制引入新成立的 **Tactical Response Protocol (TRP)**，以代碼考古與特徵測試為主，產出輕量化 Memo。

### 3. 重構全局啟動協定 (Global Startup Protocol)
- 將原先的「確認前端/後端模式」、「取得技術棧」、「解析意圖與邊界」，封裝成為僅針對 `$enterprise` 與 `$agile` 生效的 `Global Startup Protocol`。
- 輕量模式 (`$light`, `$legacy`) 不再被強迫要求填寫此類表格。

## 結論
v7.5 正式將系統從「單一的防禦型重裝甲」，進化成了可以動態裝配的武器庫。AI 現在能依照呼叫指令的不同，精準調節開發的「嚴肅度」與「防禦層級」。
