# SSDLC v6.2: 規格完整性革命 (Spec Completeness Overhaul)

## 核心目標
通過「根因分析 (RCA)」解決 AI 在規格編寫時的「滿足偏誤 (Satisficing Bias)」，確保規格覆蓋率達到 100% 穷举。

## 變更紀錄
1. **原子化規格原則 (Atomic Spec Principle)**: 一條 Spec 只能斷言一個結果。
2. **追溯矩陣 (Traceability Matrix)**: 強制產出 `PRD Section -> Spec ID` 映射表，禁止出現 "UNCOVERED" 項目。
3. **負面路徑配對 (Negative Pairing)**: 規定 Happy-Path 與 Negative-Path 的比例必須 ≥ 1:1。
4. **狀態枚舉公式 (N+1 Formula)**: 對於 N 個狀態，至少產生 N+1 條 Spec，覆蓋組合與優先級。
5. **呈報前自稽核 (Pre-Submission Audit)**: AI 在向 PM 呈報前必須先自行檢查是否有遺漏。

## 解決的痛點
- 規格遺漏了登出、權限過期或邊界場景。
- AI 的規格文件「看起來很長」，實際上卻模糊了重要的錯誤處理邏輯。
