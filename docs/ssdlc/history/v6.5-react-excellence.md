# SSDLC v6.5: React 效能卓越與文件記錄強制化 (React Excellence & Documentation Mandate)

## 核心目標
1. 整合 Vercel Engineering 的 React/Next.js 效能優化最佳實踐。
2. 建立 SSDLC 協議變更的「紀錄強制化」機制，確保開發過程中的所有工具升級與規範變更都有跡可循。

## 變更紀錄
1. **React 效能標準整合**: 建立了 `payload/.agents/standards/react-best-practices.md`。
   - 重點包含：消滅瀑布流請求 (Eliminating Waterfalls)、Bundle Size 優化、Server-Side Cache (`React.cache`)、以及防止不必要的 Client 端 Re-render。
2. **協議維護規範 (Protocol Maintenance)**:
   - 在 `copilot-instructions.md` (Section 0.7) 加入了**文件紀錄強制令**。
   - 規定爾後所有的 SSDLC 版本更迭、SKILL 或指令變更，都必須同步建立對應的歷史紀錄文件於 `docs/ssdlc/history/` 目錄。

## 解決的痛點
- 前端開發效能不一，缺乏統一的 React 現代優化指標。
- 系統演進速度快，若不強制紀錄，長期維護者將難以追溯某個開發階段採用的特定規範。
