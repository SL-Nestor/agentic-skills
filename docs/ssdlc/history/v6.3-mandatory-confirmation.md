# SSDLC v6.3: 防禦性啟動與對帳 (Mandatory Confirmation)

## 核心目標
防止 AI 在「資訊不足」的情況下「盲目開工」，增加互動式閘口確保開發環境與技術棧一致。

## 變更紀錄
1. **開發模式強制確認**: 移除 `--mode` 的預設值，若未指定則強制停止並詢問。
2. **技術棧對帳 (Tech Stack Confirmation)**: Phase 0 啟動時，AI 必須列出偵測到的庫、框架、DB，並針對缺失項目（如前端框架未指定）調用 `$stack-advisor` 進行對話。
3. **流程硬化**: 在確認 Mode 與 Stack 之前，嚴禁生成任務清單或進入 Phase 0。

## 解決的痛點
- AI 使用了錯誤的技術棧（如規格要 Vite 卻用了 Next.js）導致事後重寫成本高昂。
