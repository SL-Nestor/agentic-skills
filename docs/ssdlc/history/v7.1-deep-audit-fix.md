# SSDLC v7.1: 深度審計修復 (Deep Audit Fix)

## 核心目標
根據對 v7.0 全部核心檔案的逐行深度審計結果，一次性修復 5 項功能性衝突、3 項冗餘與 6 項不足之處，讓框架真正達到大型軟體開發的工程標準。

## 修復的功能性衝突 (Conflicts Fixed)
1. **Phase 編號斷裂**：正式定義了 Phase 4 為「架構審查與 SAST」階段，填補 Phase 跳號的空洞。
2. **Section 編號重複**：重新編號為 Section 0（角色）→ 1（流程）→ 2（反合理化）→ 3（PM 角色）→ 4（觀察性）→ 5（Artifact 映射）→ 6（Change Log），消除兩個 Section 8 與兩個 Section 9 的混亂。
3. **SSOT 自我循環**：在 Agile Mode 下，明確 PRD 必須經過 Gate P 簽核後才成為 SSOT。
4. **Plan Skill 不認識雙軌制**：在 Phase 2-3 主文件中加入 Enterprise Mode 指引（Contract-Adherence 而非 Contract-First）。
5. **Core Rules 硬編碼**：移除 `UBQ, PCM, LIC, BIL, NTF` 等特定系統名稱與 `Azure SQL` 硬限。改為引用 Source Intent Inventory 中的具名依賴。

## 新增功能 (Gaps Filled)
1. **Phase Regression Protocol (Section 0.8)**：定義了如何優雅地「退回」到前一個階段，而不是只能往前衝。
2. **Concurrency Policy (新標準文件)**：定義了多工程師/多 AI 代理人的模組隔離原則、共享模組升級路徑與合併前檢查清單。
3. **Performance Budget 驗證**：在 Phase 7-8 (Verify) 中加入效能預算量化驗證規則。
4. **Dependency Gate (Core Rules Section 3)**：引入新的第三方套件引入閘口，涵蓋授權 (License)、維護狀態、最小化三項檢查。
5. **Destructive Migration Protocol (Core Rules Section 4)**：定義了破壞性 Schema 變更的多階段遷移策略與向下相容窗口。
6. **Enterprise Ship 退出標準**：在 Phase 9-10 中加入 Enterprise 模式專屬的治理回報要求（合約對齊、交付報告、狀態更新）。

## 處理的冗餘 (Redundancies Addressed)
1. **Change Log 精簡化**：主文件只保留最近 3 個版本，舊版條目完整保存於 `docs/ssdlc/history/`。
